# 🚀 Marsy Rover

**Marsy** is a small Raspberry Pi based rover project by **AstroSheeps** 🐑✨

It started as a hands-on robotics experiment with the 4tronix M.A.R.S. Rover platform and is growing into a playful testbed for autonomous navigation, sensor-based decision making, and LLM-assisted robot control.

> Tiny rover. Big mission. Slightly suspicious sheep supervision. 🐑🛸

## 🤖 What is Marsy?

Marsy is a rover project focused on:

* ⚙️ controlling motors and steering servos;
* 📡 reading ultrasonic distance data;
* 🌈 testing RGB LEDs and keypad input;
* 🎛️ calibrating servo positions;
* 🧭 experimenting with autonomous navigation;
* 🧠 exploring how large language models can support robot behavior and debugging.

The project is intentionally practical: every idea should eventually be tested on the real rover, not only simulated.

## 🧰 Hardware

Current hardware platform:

* 🚙 4tronix M.A.R.S. Rover;
* 🍓 Raspberry Pi Zero 2 W;
* ⚙️ motor driver and steering servos;
* 📡 ultrasonic distance sensor mounted on the mast;
* 🌈 RGB LEDs;
* 🔘 keypad / onboard controls;
* 📷 camera for future navigation experiments.

More detailed hardware notes are kept in the [Wiki](https://github.com/astro-sheeps/marsy/wiki).

## ⚠️ Safety notes

This project controls real motors and servos. Be careful when testing.

Recommended precautions:

* ✋ keep fingers away from moving wheels and servos;
* 🐢 test new movement code at low speed first;
* 🧹 use a clear floor area;
* 🧱 keep the rover lifted during first motor tests;
* 🛑 stop the script immediately if the rover behaves unexpectedly;
* 🎛️ avoid forcing servos mechanically beyond their range;
* 🔋 check battery voltage before long test runs.
